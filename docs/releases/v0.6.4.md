# Kimi-GUI 0.6.4

Version 0.6.4 removes the blank or frozen-looking interval while Kimi-GUI
loads sessions and conversation history.

## A launch screen that stays alive

The splash now remains visible until the first conversation is actually ready.
Its prompt cursor repeats with a CSS-only animation while session metadata and
the latest transcript load behind it. The app no longer reveals an empty shell
between the fixed splash timer and a slow disk read.

![Animated launch state](https://github.com/kaminion/Kimi-GUI/releases/download/v0.6.4/release-0.6.4-loading-splash.png)

## Message-shaped history skeleton

Switching conversations starts a 120ms loading threshold. Warm reads replace
the transcript immediately, while slower reads show a skeleton matching the
user and assistant message layout. This keeps navigation responsive without
flashing a loader during normal cached reads.

![Conversation history skeleton](https://github.com/kaminion/Kimi-GUI/releases/download/v0.6.4/release-0.6.4-history-skeleton.png)

Additional improvements:

- Session-list placeholders cover fallback boots where the splash is
  unavailable.
- Conversation history and profile usage load concurrently.
- Stale session requests cannot replace the newly selected conversation.
- Loading and failure states include localized status and alert semantics.
- Repeating motion uses CSS opacity and transform only.
- Reduced-motion mode disables cursor movement and skeleton pulsing.
- Dark and light themes use the same final layout and hierarchy.

The motion decisions follow the performance and accessibility guidance from
[emilkowalski/skills](https://github.com/emilkowalski/skills): predetermined
loading motion stays in CSS, frequent UI transitions remain brief, and
reduced-motion preferences are respected.
