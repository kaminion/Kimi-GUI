# Kimi-GUI 0.6.5

Version 0.6.5 adds a visual Agent Skills manager and Kimi CLI slash-command
autocomplete. The release keeps Skills reusable without configuration-file
editing and makes CLI commands discoverable from the prompt.

## Manage Agent Skills visually

The new **Settings → Skills** section can:

- Add a folder containing `SKILL.md` or a single Markdown Skill.
- Install a Skill for the current user or active project.
- List compatible Kimi, Claude, Codex, and Agents discovery roots.
- Enable or disable a Skill without losing its files.
- Move an unwanted Skill to the operating system Trash.

User additions use `~/.config/agents/skills`; project additions use
`<project>/.agents/skills`. These are recommended discovery locations supported
by Kimi Code CLI. Changes are guaranteed to apply to new chats.

![Skills manager with user and project Skills](https://raw.githubusercontent.com/kaminion/Kimi-GUI/v0.6.5/docs/media/release-0.6.5-skills-manager.png)

## Complete Kimi CLI commands from the prompt

Type `/` in CLI agent mode to search the Kimi Code web-compatible command
catalog. The list also includes active built-in, user, and project Skills, with
Kimi's `/skill:<name>` naming for custom Skills.

The menu supports prefix, substring, and fuzzy matching. Navigate with the
arrow keys, complete with `Tab` or `Enter`, and close with `Esc`. Selecting an
item fills the prompt without executing it, leaving room to review the command
or add arguments.

![Kimi CLI slash-command autocomplete with an Agent Skill result](https://raw.githubusercontent.com/kaminion/Kimi-GUI/v0.6.5/docs/media/release-0.6.5-slash-autocomplete.png)

## Reliability and maintainability

- Skill discovery and mutation live behind a focused repository module rather
  than renderer filesystem logic.
- Installations stage content before an atomic rename and refuse to overwrite
  an existing Skill.
- Enable, disable, and remove operations are serialized to prevent overlapping
  UI actions from racing.
- Symbolic-link sources and paths outside known Skill roots are rejected.
- CLI command discovery falls back to active local Skills when a new or
  reconnecting session does not yet expose its daemon snapshot.
- Keyboard, accessibility, reduced-motion, discovery, and filesystem behavior
  are covered by the automated regression suite.

The existing CLI launcher already provides single-flight startup, generation
guards, fresh-port retry, health probing, and built-in-engine fallback. A
stateful circuit breaker was intentionally not added because it could delay a
user's explicit retry without improving those protections.
